## Quick start

### Configure Resources

A new installation of Kessel begins as a blank slate. To integrate a Resource (e.g. for access control), you have to configure a Resource _schema_.

Configure a basic resource that ties into RBAC for permissions. Following is an example of a Host schema configured by HBI.

<Steps>
1. Author schemas

    {/* 
    NOTE: you can write files for these and then just import them.

    e.g.

    import someRepresentation from 'src/examples/common_rep.json?raw'

    <Code code={someRepresentation> lang="json" title="host/common_representation.json"} />
    */}

    ```json title="host/common_representation.json"
    {
    "$schema": "http://json-schema.org/draft-07/schema#",
    "type": "object",
    "properties": {
        "workspace_id": {
            "type": "string"
        }
    },
    "required": ["workspace_id"]
   }    
    ```

    ```yaml title="host/config.yaml"
    resource_type: host
    resource_reporters:
      - hbi
    ```

    ```yaml title="host/reporters/hbi/config.yaml"
    resource_type: host
    reporter_name: hbi
    namespace: hbi
    ```

    ```json title="host/reporters/hbi/host.json"
    {
    "$schema": "http://json-schema.org/draft-07/schema#",
    "type": "object",
    "properties": {
        "insights_inventory_id": {
            "type": "string",
            "format": "uuid"
        }
    },
    "required": []
   }
    ```

### Set up access with RBAC

TODO: Set up a basic user with role on a workspace

In the short term, run the following commands in relations-api repo,
to add relationships to spicedb directly. 
Long term, you will do this via RBAC

```bash

# link workspace to role binding
zed relationship touch rbac/workspace:a64d17d0-aec3-410a-acd0-e0b85b22c076 t_binding rbac/role_binding:it_team_a --endpoint "$SPICEDB_IP:50051" --token foobar --insecure 

# grant permission to all principals
zed relationship touch rbac/role:inventory_hosts_admin t_inventory_hosts_read "rbac/principal:sarah" --endpoint "$SPICEDB_IP:50051" --token foobar --insecure 

# link the role binding to a role
zed relationship touch rbac/role_binding:it_team_a t_role rbac/role:inventory_hosts_admin --endpoint "$SPICEDB_IP:50051" --token foobar --insecure

# link the user sarah to the role binding
zed relationship touch rbac/role_binding:it_team_a t_subject rbac/principal:sarah --endpoint "$SPICEDB_IP:50051" --token foobar --insecure

```
